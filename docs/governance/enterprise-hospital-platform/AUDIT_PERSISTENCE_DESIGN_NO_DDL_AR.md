# تقرير تصميم استمرارية سجلات التدقيق الأمني (Audit Persistence)

يوضح هذا المستند الهيكل البرمجي المقترح لتخزين وحفظ سجلات التدقيق الأمني والعمليات الحساسة بقاعدة البيانات.

## هيكل جدول سجلات التدقيق (audit_events)
1. **معرف الحدث (event_id)**: مفتاح أساسي فريد.
2. **المستخدم والدور (actor_id, actor_role)**: لتوثيق هوية وصلاحيات منفذ العملية.
3. **نطاق العملية (facility_id, department_id)**: لتحديد مكان حدوث العملية.
4. **تفاصيل العملية وقيم الأمان (action, risk_level, status)**: لتوثيق نوع العملية ومستوى خطورتها وحالتها (تمت بنجاح، أو حُظرت).
